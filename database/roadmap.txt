-If we let it close after a read/write,  it will use more CPU cycles, but in return, is safer and it shouldn't cause a memory leak.
-If we kep the databse file open after a read/write, you wouldnt have to close it, so that would use less CPU cycles in theory, but if the program ends or the program crashes, the file isnt closed and thats a memory leak. In addition to that, checking if the file is already open will take up CPU cycles..
